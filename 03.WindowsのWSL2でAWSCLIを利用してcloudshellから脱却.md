## Phase 1：WSL2 (Ubuntu) のスクラップ＆ビルド
1. 既存のUbuntuを跡形もなく消去（Destroy）

    `wsl --unregister Ubuntu`

2. 最新のUbuntu LTSをクリーンインストール（Create）

    `wsl --install -d Ubuntu`

```
※インストール完了後、自動的にUbuntuのターミナルが開きます。
 「Enter new UNIX username:」と聞かれるので、好きなユーザー名とパスワードを設定してください。
 ```

## Phase 2：AWS CLI v2 のインストール
1. パッケージ一覧の更新と、解凍ツール(unzip)のインストール

    `sudo apt update && sudo apt install -y unzip curl`

2. AWS CLI v2のインストーラーをダウンロード
    ```bash
    curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
    ```
3. 解凍してインストール実行
    ```bash
    unzip awscliv2.zip
    sudo ./aws/install
    ```
4. バージョン確認（インストール成功のヘルスチェック）

    `aws --version`

    `→ 「aws-cli/2.x.x Python/...」と表示されれば完璧！`

## Phase 3：IAMアクセスキーの発行（AWSコンソール画面）
    ブラウザで AWSマネジメントコンソール にログインする。

    上部の検索バーで 「IAM」 と検索し、IAMダッシュボードを開く。

    左側のメニューから 「ユーザー」 をクリックし、対象のユーザー（またはTerraform用に作った管理者ユーザー）をクリックする。

    「セキュリティ認証情報」 タブを開く。

    下にスクロールして 「アクセスキーを作成」 をクリックします。

    ユースケースで 「コマンドラインインターフェイス (CLI)」 を選択し、チェックを入れて「次へ」。

    「アクセスキーを作成」をクリックすると、以下の2つが表示される。

    アクセスキー ID （例: AKIA...）

    シークレットアクセスキー （※ここでしか表示されない！）

    ※この画面を開いたままにするか、「.csv ファイルをダウンロード」しておく。

## Phase 4：ローカル環境とAWSの接続（aws configure）
1. 初期セットアップコマンドを実行

    `aws configure`

    AWS Access Key ID [None]: （Phase 3で取得した アクセスキーID をペースト）
    AWS Secret Access Key [None]: （Phase 3で取得した シークレットアクセスキー をペースト）
    Default region name [None]: ap-northeast-1  （※東京リージョンを指定）
    Default output format [None]: json  （※出力形式はjsonが見やすい）

## Phase 5：最終疎通確認（Ping）
